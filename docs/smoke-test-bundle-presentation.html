Architecture

Two-Tier Testing

Separate fast validation from comprehensive verification.

CLI Tests ~30 sec

  • No LLM required
  • Tests CLI commands
  • Validates configuration
  • Checks resources exist
  • CI-friendly with --skip-llm

LLM Tests ~3-5 min

  • Requires provider API key
  • Tests actual AI responses
  • Multi-provider validation
  • Tool execution via LLM
  • Agent delegation chains

Key Innovation

Token Marker Technique

Solving LLM non-determinism with deterministic markers.

# Provider responds: Just echo a marker amplifier run 'Reply with exactly: SMOKE_TEST_OK' # Tool execution: LLM must use bash tool amplifier run 'Use bash to run: echo TOOL_OK' # Agent delegation: LLM must delegate to sub-agent amplifier run 'Delegate to explorer: Reply with AGENT_OK'

If marker appears = PASS
If marker missing = FAIL

Simple, reliable, no fuzzy matching needed.

Design Decisions

Smart Defaults

on_error: continue
Collects ALL failures instead of stopping at the first. You see the complete picture, not just the first problem.
--skip-llm Mode
CI-friendly mode that skips provider tests. No API keys needed for basic validation.
Auto-Detection
Providers detected via environment variables. Unconfigured providers are skipped, not failed.
Declarative Recipe
700 lines of YAML beats imperative scripts. Easy to read, extend, and maintain.
Usage

How to Run

Install once, run anytime.

# Install the bundle $ amplifier bundle add git+https://github.com/samueljklee/amplifier-bundle-smoke-test --app # Run via conversation (with LLM tests) > "Run smoke tests" # Run via conversation (CLI only, no API calls) > "Smoke test --skip-llm" # Run via direct CLI invocation $ amplifier tool invoke recipes \ operation=execute \ recipe_path=@smoke-test:recipes/smoke-test.yaml

Summary

Key Takeaways

1. Self-Hosting Wins
Using Amplifier to test Amplifier creates a virtuous cycle of quality.
2. Declarative > Imperative
YAML recipes are easier to read, extend, and maintain than shell scripts.
3. Skip Modes Matter
--skip-llm serves CI pipelines. Full mode serves local development.
4. Token Markers Work
Deterministic markers solve LLM testing without fuzzy matching.
