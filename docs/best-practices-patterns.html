Sub-Agent Strategy

When to delegate to sub-agents

Specialized Tasks
Bug hunting, security review, architecture design—delegate to agents with specialized context.
Fresh Context
When your context is bloated, spawn a sub-agent with just what it needs. Clean slate.
Parallel Work
Need to research while implementing? Spawn parallel sub-agents. Work concurrently.
Context Discipline

Don't load everything yourself

  • ✓ Use targeted reads—specific files, not entire directories
  • ✓ Let sub-agents fetch what they need—don't pre-load
  • ✓ Use grep and glob to find before reading
  • ✓ Context files in bundles load automatically—design them well
  • ✗ Don't read 50 files "just in case"
Repeatable Workflows

When to use recipes

Multi-Step Tasks
Research → Design → Implement → Test? That's a recipe. Sequential agent handoffs with context accumulation.
Human-in-Loop
Need approval before deployment? Recipes support approval gates. Agent waits for your decision.
Resumable Work
Long-running task might be interrupted? Recipes checkpoint automatically. Resume where you left off.
Cost Optimization
Use Claude Haiku for simple steps, Opus for complex ones. Recipes let you specify model per step.
Coordination Patterns

Multi-agent best practices

Coordinator + Specialists pattern: Parent agent coordinates, delegates specific work to specialized sub-agents. Avoids context explosion.
  • ✓ Parent tracks high-level plan
  • ✓ Sub-agents get minimal, focused context
  • ✓ Results flow back up for integration
  • ✗ Don't pass entire history to every sub-agent
Safe Experimentation

Shadow environments for testing

Test Before Pushing
Shadow environments let you test local changes in isolation. No polluted environments.
Multi-Repo Changes
Testing core + foundation together? Shadow snapshots multiple repos simultaneously.
Verify Correctness
Don't assume your local code is being used—verify with snapshot commits.
What NOT To Do

Common mistakes

  • ✗ Over-planning — Don't spec for 3 hours, code for 10 minutes
  • ✗ Insufficient delegation — Trying to do everything yourself
  • ✗ Ignoring errors — "Let's move on" when tools fail
  • ✗ Context dumping — Loading entire repos into context
  • ✗ No validation — Implementing without reviewing
