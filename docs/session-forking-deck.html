Session Forking

Branch your conversations at any point.
Explore alternate approaches without losing your place.

January 2026
The Problem

Conversations are linear.
Thinking isn't.

"What if..."
You're 20 turns in and wonder what would have happened with a different approach. Start over?
Exploration Anxiety
Afraid to try risky ideas because you might break the good work you've already done.
Lost Context
Starting a new session means re-explaining everything. The AI doesn't remember where you were.
What if conversations worked like git branches?
The Solution

Fork any session at any turn

Create a branch from any point in your conversation history.
The fork inherits all context up to that turn, then diverges.

How It Works

Git-style branching for conversations

Turn 2: Discusses OAuth approach
Turn 3: Fork point
Turn 4: Implements OAuth

Full Lineage

Every fork knows where it came from

Tracked Metadata
parent_id - Which session this forked from

forked_from_turn - The exact turn number

forked_at - Timestamp of the fork

Usage: In-Chat

Fork without leaving your conversation

# Just type /fork in any session /fork # Select a turn, fork is created # You stay in current session - fork runs in background
Key behavior: The /fork command creates the fork but keeps you in your current session. No context switch. Resume the fork later when you're ready.
Use Cases

When to fork

A/B Testing Ideas
Try two different architectural approaches from the same starting point. Compare results.
"What if we used Redis instead?"
Safe Exploration
Fork before trying something risky. If it breaks, your original session is untouched.
"Let me experiment without fear"
Revisit Decisions
20 turns later, realize an early decision was wrong. Fork from that turn, try again.
"I should have chosen differently at turn 5"
Under the Hood

How forking works

  • ◆ Events.jsonl rewritten with new session_id
  • ◆ Parent reference preserved in metadata
  • ◆ All context up to fork point inherited
  • ◆ Fork is a fully independent session from that point
  • ◆ Tree view shows relationships across all sessions
Related: Sub-Agent Inheritance

Sub-agents inherit context and hooks

Context Inheritance

# Pass conversation history: inherit_context: "recent" inherit_context_turns: 5 # Modes: none | recent | all

Hook Inheritance

# Pass active hooks to sub-agents: inherit_hooks: true # Sub-agents get same logging, # notifications, and policies
Sub-agents no longer start fresh. They pick up where you left off and behave consistently with parent session policies.
