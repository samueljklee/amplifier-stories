How It Works

Integration Layer

Seamless communication between UI and Amplifier Core.

# Browser-session lifecycle Each browser tab = ONE WebSocket = ONE Amplifier session // Session created when browser connects WebSocket connection → Backend creates session User sends message → Amplifier processes prompt Response streams → Frontend updates in real-time Browser closes → Session saved to disk

WebSocket Communication

✓ Event-driven architecture
✓ Server-sent events (SSE) for streaming
✓ Automatic reconnection
✓ Auth token validation

Python Integration

✓ FastAPI WebSocket endpoints
✓ Amplifier Core subprocess orchestration
✓ Session lifecycle management
✓ Storage in .nexus-data/sessions/
