Brian Krabach's insight: using AI to build tools that improve AI usage creates compounding returns. Most people spend their AI gains. Brian was compounding them. This became Amplifier.

IMPLICATIONS

The Uncomfortable Implications

  1. Senior engineers aren't worth the hassle — if they're high-maintenance. Skill without low-overhead collaboration is attention-negative.
  2. Every management layer is an attention tax. Each layer filters, translates, and delays. The tax compounds.
  3. Hire for imagination and low overhead. Technical skill is increasingly table stakes. Imagination and self-direction are the scarce resources.
  4. The skill gap is imagination, not technique. Knowing how to code matters less than knowing what to build and how to direct.

"Software Dentists"

Routine technical work. Automatable. In trouble.

"Obsessive Graybeards"

Deep craft + imagination. Amplified. Golden age.

DESIGN

What Makes Something Agent-Shaped?

Works Well With Agents

  • Text-native formats
  • Composable, pipeable
  • Git-versioned state
  • HTML for presentation
  • Simple, predictable state
  • Schema-describable interfaces

Struggles With Agents

Visual-only interfaces (no text API)

Complex multi-step toolchains

Stateful sessions with side effects

Binary formats

Proprietary protocols

Authentication-heavy flows

"This is like the CLI to GUI shift — we need new primitives for a new interaction model."

LITMUS TEST

The Eigenvector Test

Six criteria that predict whether a technology works well with agents:

Text-native

Readable and writable as plain text

Zero-SDK

No library install required

Composable

Pipeable, chainable, mixable

Schema-describable

Structure fits in a system prompt

Deterministic shape

Predictable output structure

Append-friendly

Add without rewriting

Traditional alternatives fail: React fails zero-SDK. Databases fail composable. Binary formats fail text-native. GraphQL fails schema-describable.
