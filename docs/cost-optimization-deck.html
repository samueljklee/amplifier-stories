Bonus
Token-Safe Git Status

Prevent accidental token DoS

The Problem
Large repos with many uncommitted changes can produce 100k+ token git status output.

Status context hook injects this every turn. One bad repo = context overflow.
The Fix
Tier-based filtering automatically truncates large status output:

Tier 1: Full output (small repos)
Tier 2: Truncated with summary
Tier 3: Counts only (huge repos)

Implementation: hooks-status-context PR "fix: prevent token consumption DoS"
