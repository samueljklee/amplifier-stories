The Problem

Support requests get lost.
Status updates are tedious.

When something breaks, you context-switch to file an issue. Then you forget to update your team on what you're working on. Information silos form.

The Insight

Your session already has
all the context.

Errors, steps taken, what was attempted. It's all there. Why re-type it into a GitHub issue? Why manually update a status doc?

The Solution

Three capabilities, one bundle

🤝
Support Intake
Say "I need help with..." and the bundle files a GitHub issue with full session context. Checks for duplicates first.
📊
Team Status New
Automatically tracks what you're working on. Updates team-status.md with your recent progress. No manual updates needed.
🎬
Story Submission
Say "make an amplifier story" to generate a presentation deck from your session and submit it as a PR.

Privacy Controls

You control what's shared

Configure tracking behavior in ~/.amplifier/team-tracking.yaml

enabled: true throttle_minutes: 5 exclude_repos: - personal-* - experiments/* detail_level: "summary" # or "detailed" share_errors: false # don't include error context
  • ✓ Disable entirely with one flag
  • ✓ Exclude specific repos by pattern
  • ✓ Control detail level of updates
  • ✓ Opt out of error context sharing

What's in the Bundle

5 recipes, 1 hook, 2 behaviors

Component Type Purpose
check-gh Recipe Verify GitHub CLI authentication
extract-issue-full Recipe Extract issue context from session
find-duplicates Recipe Search existing issues for matches
file-issue Recipe Create GitHub issue with context
update-team-status Recipe Update team-status.md file
team-status Hook Fires on turn:end (throttled)
support-intake Behavior Triggers on "I need help..."
story-submit Behavior Triggers on "make a story..."
