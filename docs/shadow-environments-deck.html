Platform Feature

Shadow Environments

OS-level isolated containers for safe testing.
Test your changes before you push.

January 2026
The Problem

Testing local changes is risky

Polluted Environment
Your local machine has years of accumulated state. "Works on my machine" is the oldest lie.
Cross-Repo Dependencies
Changing amplifier-core? Need to test it with amplifier-foundation. Local symlinks are fragile.
Push to Test
Often the only way to really test is to push and wait for CI. Slow and public.
The Solution

Shadow environments

Isolated containers that snapshot your local repos.
Test exactly what you have, in a clean environment.

How It Works

Git URL rewriting + embedded Gitea

Your Local Repo
~/repos/core
→
Gitea Server
Snapshot
→
Container
Isolated Test
Git is configured to rewrite specific GitHub URLs to the embedded Gitea server. Your local changes appear as the "real" repo inside the container.
Usage

Simple operations

# Create shadow with local source shadow create --local ~/repos/amplifier-core:microsoft/amplifier-core # Run commands inside shadow exec <id> "uv pip install git+https://github.com/microsoft/amplifier" shadow exec <id> "amplifier run 'Hello from shadow'" # Your local amplifier-core is used! # Clean up shadow destroy <id>
Key Features

What you get

  • ✓ Full git history preserved (pinned commits work)
  • ✓ Multiple local sources in one shadow
  • ✓ API keys auto-passed from environment
  • ✓ File injection/extraction for iterating
  • ✓ Diff view of all changes made
  • ✓ Security hardened containers
Multi-Repo

Test changes across repositories

# Testing amplifier-core + amplifier-foundation together shadow create \ --local ~/repos/amplifier-core:microsoft/amplifier-core \ --local ~/repos/amplifier-foundation:microsoft/amplifier-foundation # Inside the shadow: # - amplifier uses real GitHub # - amplifier-core uses YOUR local snapshot # - amplifier-foundation uses YOUR local snapshot shadow exec <id> "uv tool install git+https://github.com/microsoft/amplifier" # Dependencies resolve to your local code!
Trust But Verify

Know your local code is actually used

Snapshot Commits
Create and status operations return the exact commit hash captured from your local repo.
Install Output
When pip/uv installs, it shows the resolved commit. Match against snapshot_commits to verify.
Don't assume—verify. If the commits match, your local code is definitely being used.
