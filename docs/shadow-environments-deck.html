Usage

Just ask naturally

# In any Amplifier session: "Create a shadow environment and test if my-library installs correctly" "Test my local changes to auth-service in an isolated container" "Verify this works on a fresh machine"

The shadow-operator agent handles container creation, source snapshotting, and test execution.

Programmatic Control

Full tool integration

# Create with local sources shadow(operation="create", local_sources=[ "~/repos/my-lib:myorg/my-lib" ]) # Run commands inside shadow(operation="exec", shadow_id="...", command="uv pip install git+https://github.com/myorg/my-lib") # Check what changed shadow(operation="diff", shadow_id="...") # Clean up shadow(operation="destroy", shadow_id="...")
Multi-Repo Development

Test cross-repo changes together

# Snapshot multiple repos shadow(operation="create", local_sources=[ "~/repos/amplifier-core:microsoft/amplifier-core", "~/repos/amplifier-foundation:microsoft/amplifier-foundation" ]) # Install amplifier - uses YOUR local amplifier-core shadow(operation="exec", command="uv tool install git+https://github.com/microsoft/amplifier") # Everything else fetches from real GitHub
Key: Only repos you specify redirect to your snapshots. Everything else uses real GitHub - no isolation theater.
