Recipes & Workflows

Declarative multi-step agent coordination

January 2026
The Problem

Why recipes exist

Multi-Step Tasks
Complex workflows involve multiple agents in sequence. Orchestrating them manually is error-prone.
Repeatability
You run the same workflow repeatedly. Writing it as code makes it consistent and versioned.
Interruptions
Long-running work gets interrupted. You need to resume exactly where you left off.
Use Cases

Perfect for

  • ✓ Multi-step tasks requiring different agents sequentially
  • ✓ Workflows that will be repeated (worth encoding as YAML)
  • ✓ Human approval checkpoints between phases
  • ✓ Work that might be interrupted and needs resumption
  • ✓ Context accumulation across agent handoffs
For simple tasks, direct agent delegation is faster.
Anatomy

YAML workflow definition

name: code-review-workflow description: Review and deploy code changes steps: - id: analyze agent: foundation:explorer output: analysis prompt: "Analyze changes in {{file_path}}" - id: review agent: foundation:zen-architect prompt: "Review code quality: {{analysis}}" # Approvals configured in staged mode stages: - name: review steps: [analyze, review] approval: required: true prompt: "Approve deployment?" - name: deploy steps: - id: deploy type: bash command: "deploy.sh {{file_path}}"
Capabilities

What you get

Context Accumulation
Use output field to create named variables (e.g., output: analysis) then reference directly: {{analysis}}
Approval Gates
Pause for human review at critical checkpoints. Agent waits for your decision.
Automatic Checkpointing
Every step completion is saved. Resume interrupted workflows seamlessly.
Provider Selection
Specify provider and model per step. Use Haiku for simple steps, Opus for complex ones.
Parallel Execution
Use parallel: N field on foreach loops to run iterations concurrently. Speed up independent work.
Conditional Logic
Skip steps based on conditions. Branch workflows dynamically.
Human-in-Loop

Approval checkpoints

Agent: Research
↓
Agent: Design
↓
⏸ Approval Gate
↓ human approves
Agent: Deploy
Critical checkpoints. Review before deployment, approve before destructive operations, inspect before committing.
Fault Tolerance

Resume where you left off

# Recipe interrupted at step 3? No problem. amplifier run "resume recipe session recipe_20260121_abc123" # Or via in-session command: > Resume the recipe workflow # Picks up exactly where it stopped # All previous step results preserved # Context accumulation intact
Long-running workflows. Close your laptop, go to meetings, come back and resume. The recipe waits.
Real Example

Feature announcement workflow

steps: - id: research agent: foundation:explorer model: claude-haiku # Fast & cheap prompt: "Gather feature details" - id: draft agent: writer model: claude-sonnet-* # Good writing prompt: "Draft announcement: {{steps.research.result}}" - id: review type: approval_gate message: "Review draft before posting" - id: post type: bash command: "post-to-teams.sh {{steps.draft.result}}"
