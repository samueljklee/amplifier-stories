Guiding Principles

Core philosophy

  • → Mechanism, not policy — Kernel provides capabilities, not constraints
  • → Ruthless simplicity — As simple as possible, but no simpler
  • → Event-first — Everything emits events for observability
  • → Composable by design — Mix and match capabilities freely
  • → The center stays still — Kernel stability enables edge innovation
Design Thinking

Bricks & studs (LEGO model)

Each module is a self-contained "brick." Interfaces are "studs" where bricks connect. You can regenerate any brick independently without breaking the system.
Stable Interfaces
Provider protocol, Tool contract, Hook API—these stay stable while implementations evolve.
Independent Evolution
Swap providers, update tools, add hooks—all without touching the kernel.
Litmus Test
"Could two teams want different behavior?" If yes, it's policy → Module, not kernel.
