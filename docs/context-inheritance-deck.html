One Config, Automatic Context

Bundle Config
# agents/explorer.yaml
inherit_context: recent
inherit_context_turns: 3
Task Call

  agent="explorer",
  instruction="Find auth bug"
)

Child automatically receives last 3 turns of conversation

Configuration

Three Inheritance Modes

Mode Behavior Use Case
none Child starts fresh Privacy-sensitive tasks, stateless ops
recent Pass last N turns Most common ‚Äî recent context without overflow
all Pass entire history Full conversation context needed

Default is none ‚Äî opt-in only, no surprise context sharing

Architecture

How It Works

Task Tool

Task tool reads inherit_context from bundle config
Extracts last N turns from parent's ContextManager
Sanitizes to user/assistant roles only
Passes to child via set_messages()

No kernel changes ‚Äî uses existing ContextManager API

Philosophy

Policy at the Edges

01
Opt-in Only
Default to none ‚Äî agents must explicitly request context inheritance
02
Turn-based, Not Message-based
More intuitive ‚Äî "last 3 exchanges" vs "last 6 messages"
03
Sanitized Messages
Only user/assistant roles passed ‚Äî no system prompts or tool calls leak
04
Resume Compatible
Ignores parent context when resuming ‚Äî uses persisted state instead
