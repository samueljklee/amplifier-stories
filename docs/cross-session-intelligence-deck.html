Phase 1 ‚Äî Tag Foundation

Give sessions a vocabulary

A JSON-backed TagStore that lets sessions carry metadata. Tags are normalized, filterable, and visible in the sidebar.

# Tag sessions manually /tag auth-refactor /tag caching performance # Filter sessions by tag #auth ‚Üí finds all auth-tagged sessions #performance ‚Üí finds performance sessions # Sort by tag groups /sort tag ‚Üí groups sidebar by first tag

Key decisions

Tags normalized (lowercase, stripped). #prefix search matches tags only. Sidebar shows up to 2 tags inline per session.

Phase 4 ‚Äî /project ask

Ask questions across all your sessions

The AI draws on your entire project history ‚Äî every session, every tag, every decision ‚Äî to answer questions about your work.

Powered by Sonnet for quality synthesis. 5-minute answer cache. Builds context from project name, top 10 tags, last 15 sessions with dates and tags.
Phase 4 ‚Äî Full Command Suite

Four intelligence commands

/project ask <question>

Free-form Q&A across your entire project. Ask anything ‚Äî status, history, decisions, patterns. LLM references specific sessions by date and ID.

/project search <query>

Search transcript content across all sessions. Reads last 64KB per session, returns up to 20 results sorted by recency with context snippets.

/project attention

Identifies stalled work, frequent topics indicating issues, and sessions that ended without resolution. Prioritized and actionable.

/project weekly

Weekly summary grouped by theme ‚Äî not chronologically. Key outcomes, decisions, and blockers surfaced automatically.

Concurrent Sessions

SessionHandle + registry pattern isolates per-session state. Each handle carries its own streaming callbacks and token counters. Zero cross-talk between concurrent sessions.

Answer Caching

5-minute cache keyed by MD5(project + question). Same question in the same project returns instantly. Different phrasing gets a fresh answer.

Bonus Feature

/commit ‚Äî one command, full workflow

/commit ‚Üí git add -A ‚Üí generates diff summary ‚Üí LLM writes conventional commit message ‚Üí git commit ‚Üí git push /commit nopush ‚Üí same flow, skips push /commit info ‚Üí dry run: branch, staged, modified, untracked
Why it matters

Runs in the active session's project directory ‚Äî not the TUI launch dir. Generates conventional commit format (feat:, fix:, refactor:). Includes Amplifier co-author trailer.

Combined with /project ask, you can ask "what did I change today?" then commit it all in one flow.
